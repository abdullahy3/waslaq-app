## 🚨 Flutter-Specific Non-Negotiable Rules

### File editing
1. **All Dart files require FULL REWRITES** via `Filesystem:write_file`
   - `str_replace` corrupts Dart syntax — do NOT use it for Dart files
   - Never partial-edit `.dart` files

### After code generation — ALWAYS run these
2. After `@RoutePage()` annotations added/changed:
   ```bash
   dart run build_runner build --delete-conflicting-outputs
   ```
3. After JSON translation key changes:
   ```bash
   dart run slang
   ```
   This regenerates `lib/gen/strings.g.dart`

### iOS pbxproj editing
4. **NEVER use Python `str.replace()` on pbxproj files**
   - Use line-number-based insertion only
   - After every edit: `plutil -lint ios/Runner.xcodeproj/project.pbxproj`
   - If corrupted: `git checkout ios/Runner.xcodeproj/project.pbxproj`

### Dart async patterns
5. `catchError((_) => null)` **crashes at runtime on non-nullable Futures**
   - Use: `.then((r) => variable = r).catchError(...)` pattern instead

### State management
6. Tabs in TabBarView: use `ConsumerWidget` per tab — NOT parent WidgetRef in layout phase (causes infinite rebuild loop)
7. `ElevatedButton` in Row: must have `minimumSize: const Size(0, 0)` to prevent crash

### Theme colors
8. After rewriting `app_colors.dart` as instance-based:
   - Remove `const` from ALL widget constructors that use `context.colors`
   - `const` is invalid when color depends on context

## 🔔 Push Notifications Architecture

### FCM payload (backend sends this)
```json
{
  "data": {
    "title": "واصلك",
    "body": "...",
    "link": "/r/community/comments/postId"
  }
}
```
**NO `notification:` key** — data-only prevents Android double-display

### Routing flow for notification taps
1. Foreground: `FirebaseMessaging.onMessage` → `_showLocalNotification(payload=data['link'])`
2. Background tap: `onDidReceiveNotificationResponse` (flutter_local_notifications) → `_navigateToLink(link)`
3. **NOT** `onMessageOpenedApp` — that only fires for `notification:` key payloads

### `_navigateToLink(link)` pattern
```dart
// Regex-match path → explicit AutoRoute objects:
/r/[slug]/comments/[id] → PostDetailRoute(...)
/u/[userId]             → UserProfileRoute(...)
/orders                 → OrdersRoute()
/vendor-dashboard       → VendorDashboardRoute()
/vendors/[slug]         → VendorProfileRoute(slug: slug)
```

### Bell badge (BellIconWithBadge)
- `ConsumerStatefulWidget` + `WidgetsBindingObserver`
- `initState`: registers on `notification_bus.dart` + lifecycle observer
- `didChangeAppLifecycleState(resumed)`: `ref.invalidate(notificationsProvider)`
- `dispose`: removes both listeners
- Triggered by: FCM `onMessage` + `_handleNotificationTap` → `notifyRefreshListeners()`

### NotificationsScreen
- On mount (`initState`): calls `markAllNotificationsRead()` + `ref.invalidate(notificationsProvider)`
- Badge resets to 0 when screen opens

## 🛍️ Commerce Integration

### Cart sync
- `CartRepository.getOrCreateCart` calls `GET /store/customer/active-cart` first
- Creates new cart only if no active cart exists — prevents desync across devices

### Inventory
- Sum across ALL variants — pick variant WITH inventory as primary
- `allVariantIds: List<String>` field — `updateProduct()` sends ALL variant IDs
- Backend uses raw SQL for inventory block (inventoryService unreliable)

### Orders
- `GET /store/custom/orders` — joins vendor_sub_order for real fulfillment status
- Total from `order_summary.totals` JSONB
- `vendor_sub_order.status = 'shipped'` → shows as Delivered

## ⚡ Rendering / Scroll Performance (root-caused 2026-06-22, profile-traced on A55)
- **Impeller is DISABLED** via `AndroidManifest.xml` meta-data `io.flutter.embedding.android.EnableImpeller=false`. Flutter 3.44.1 Impeller (Vulkan) rebuilt the text glyph atlas EVERY frame (`CreateGlyphAtlas` 137×/137 frames) on this bilingual AR/EN app + emitted huge saveLayers → raster thread saturated → "freeze-and-snap" scroll. Measured via VM-service timeline (`getVMTimeline`): raster busy 3075ms→1046ms, saveLayer total 1586ms→10ms, worst frame 196ms→10ms, median 11.3ms→3.4ms. Symptom was RASTER-bound (`1.raster` thread), NOT UI/Dart thread (UI was ~570ms, fine). Re-enable Impeller only after upgrading Flutter + re-profiling.
- Diagnosis method (reuse): `flutter run --profile -d R5CX7289WNF`, drive scroll with `adb shell input swipe`, `curl <vmservice>/getVMTimeline`, pair B/E events per thread. gfxinfo is USELESS for Flutter (bypasses HWUI → "0 frames").
- **High refresh rate (90/120Hz) enabled app-wide** via `flutter_displaymode: ^0.6.0` — `main()` calls `FlutterDisplayMode.setHighRefreshRate()` (Android-only guard, try/catch). Picks panel's highest mode at current res; 60Hz-only devices stay 60 (graceful). Verified on A55: `dumpsys display` shows `frameRateOverride uid=...120.0` + measured ~114fps during scroll (was 60). iOS ProMotion already handled by `CADisableMinimumFrameDurationOnPhone` in Info.plist. NOTE: high refresh halves the frame budget (120Hz=8.3ms) — only viable because the Impeller raster fix landed first.
- Secondary fixes shipped same day: right-sized `memCacheWidth` (store logo 600→110, banner→330, product→480, category→360, bottom-nav avatar uncapped→80) to cut GPU texture churn; `PostCard`→StatelessWidget + isolated `_BookmarkButton` ConsumerWidget (one bookmark toggle no longer rebuilds every card); explore header rebuilt as `NestedScrollView` + `SliverAppBar`(floating/snap) + pinned `SliverPersistentHeader`(TabBar, wrapped in RepaintBoundary) + `TabBarView`(swipe-disabled) + `_KeepAlive`, replacing the old SizeTransition/NotificationListener that relayed out the viewport each scroll-direction flip.
